# WIF File Import

## Standard

The platform targets the **WIF 1.1 specification** (April 20, 1997) as the authoritative standard. The spec is located at `docs/standard/standard-wif1-1.txt`.

Sample WIF files for testing are located at `docs/samples/`.

## Import Behavior

- WIF files are **always imported**, even if they contain errors or deviate from the spec
- Errors and warnings are displayed to the user after upload
- Features that cannot function due to file problems are **disabled** for that file with a clear explanation
- The platform does not modify the uploaded WIF file

## Linting Rules

The linter validates the uploaded file against the WIF 1.1 spec, checking:

- Presence of required sections: `[WIF]` and `[CONTENTS]`
- Required keys within each declared section (`Version`, `Date`, `Developers`, `Source Program` in `[WIF]`; `Shafts`, `Treadles` in `[WEAVING]`; `Threads` in `[WARP]` and `[WEFT]`)
- All sections declared as `true` in `[CONTENTS]` are present in the file
- No duplicate key names within a section (flagged as a warning, not fatal)
- Data type validation for each key (integer, real, boolean, string, RGB triplet, symbol)
- Valid index references (e.g. color table indices within declared palette range)
- Consistent thread counts (threading entries do not exceed declared warp thread count, etc.)
- Obsolete keys noted as informational warnings, not errors
- Private sections (`[PRIVATE <SourceID> <SectionName>]`) are recognized and passed through without error

## Feature Availability from WIF Data

| Feature | Required WIF Data |
|---|---|
| Treadle-tracking project | `[TREADLING]` section present |
| Lift-tracking project | `[LIFTPLAN]` section present |
| Tie-up sheet | `[TIEUP]` section present |
| Color preview | `[COLOR TABLE]` and `[COLOR PALETTE]` present |
| Threading diagram | `[THREADING]` section present |
| Drawdown | `[THREADING]` + `[TIEUP]` + (`[TREADLING]` or `[LIFTPLAN]`) |

If a required section is absent, the dependent feature is disabled and the reason is noted in the import report.

## Auto-Computation of Liftplan from Treadling

Computing a liftplan from `[TREADLING]` + `[TIEUP]` when `[LIFTPLAN]` is absent (and vice versa) is a **Phase 2 feature**. In the initial build, the platform uses only what is explicitly present in the WIF file.

## Compatibility Tracking

When a WIF file produces warnings or errors, the platform:

1. Prompts the user to identify the software and version used to generate the file
2. Stores the user's response alongside the file's diagnostic report in admin records
3. Admins can review this data to identify patterns of software incompatibility

## Third-Party Software Notes

The platform expects WIF files to be generated by third-party weaving software. Known software that produces WIF files includes:

- TempoWeave
- Fiberworks PCW (co-developed the WIF standard)
- WeavIt
- WeavePoint
- ProWeave
- SwiftWeave
- Patternland Weave Simulator
- AVL programs

As compatibility issues are discovered through user reports, the platform documentation should be updated to note which software versions produce non-standard output and how those files are handled.
